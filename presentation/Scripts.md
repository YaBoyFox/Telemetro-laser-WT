Adonai:Buen día como mencionó mi compañero mi nombre es (tu nombre) y yo me encargué de la parte de desarrollo de la aplicación en Android ya que al ser un proyecto de programación debe tener un programa, en este caso la primera opción para el desarrollo del programa fue usar MIT app designer, un programa para crear las aplicaciones mediante uniones de bloques, aunque la app fue funcional y también se puede encontrar en el GitHub del proyecto la aplicación no mostraba los resultados de las mediciones a pesar de conectarse perfectamente al módulo Bluetooth, al comentarlo con mi compañero descubrimos que hubo un “error de comunicación” ya que la aplicación solo funciona como un receptor, recibiendo los resultados de las mediciones solamente, mientras que mi compañero en el arduino hizo un código que en la terminal funciona como transmisor, enviando un comando en el que el circuito se activa y envía la medición como respuesta, así que para resolver eso intentamos varias 2 cosas, la primera siendo que mi compañero cambió el código del arduino por uno en el que el código funciona de manera automática y el HC-05 se encarga de enviar simplemente las mediciones a la aplicación, no funciono así que la otra opción pero mas difícil fue usar otro programa, en este caso fue Android Studio, al ser más difícil que MIT se tuvo que hacer entre ambos para tener la aplicación funcional, que también estará en el repositorio junto a la otra versión del código del Arduino.

Patricio:Bien, después de la explicación de mi compañero llegamos a este punto, donde tengo en mis manos el producto final, aunque pudieron haberse hecho algunos cambios como hacer que la app sea una transmisora y no simplemente receptora de la información el uso de Android studio fue lo suficientemente difícil para que esta versión de la aplicación sea lo suficientemente convincente, este diseño estuvo pensado desde un inicio para no depender de estar conectado a una computadora para funcionar, dándole cierta portabilidad, bien, al darle corriente al circuito lo primero que hacemos es conectar el módulo al teléfono, después de eso entramos a la aplicación y vinculamos el módulo con la aplicación para empezar a recibir los datos de medición, en este caso son entre intervalos de 2 segundos, hay que tener en cuenta que puede hacer cierto margen de error ya que es posible que la velocidad de procesamiento del Arduino no se compare a la velocidad de la luz del láser del módulo, haciendo que cuando el láser rebote y reciba la señal en el tiempo que reacciona el Arduino ya pasó cierto tiempo que agrega ese margen de error, con un micro controlador más potente este margen de error se puede eliminar fácilmente.
